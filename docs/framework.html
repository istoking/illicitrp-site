Framework and Tech Overview

IRP is built on a modern FiveM stack with a focus on stability, performance, and long-term maintainability.

Core philosophy

  • Production over prototypes: features should be reliable and low impact.
  • Clear ownership: resources have defined owners and documented configs.
  • Predictable behavior: systems should be consistent so roleplay feels fair.

Core framework

Framework: QBCore-based stack with curated resources and consistent UI patterns.

Client performance

Optimized loopsLow tick usage where possible and event-driven behavior.
Clean UIResponsive NUI with minimal redraw and consistent styling.
Stable dependenciesVersion-locked resources and clear load order.

Systems you will see in city

  • Police and emergency operations (MDT, dispatch, patrol workflows).
  • Criminal economy with progression and risk.
  • Vehicle systems, garages, and class-based balance.
  • Quality-of-life UI for roleplay and immersion.
